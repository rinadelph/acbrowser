# Commands

## Core

```bash
acbrowser open <url>              # Navigate (aliases: goto, navigate)
acbrowser click <sel>             # Click element (--new-tab to open in new tab)
acbrowser dblclick <sel>          # Double-click
acbrowser fill <sel> <text>       # Clear and fill
acbrowser type <sel> <text>       # Type into element
acbrowser press <key>             # Press key (Enter, Tab, Control+a) (alias: key)
acbrowser keyboard type <text>    # Type at current focus (no selector needed)
acbrowser keyboard inserttext <text>  # Insert text without key events
acbrowser keydown <key>           # Hold key down
acbrowser keyup <key>             # Release key
acbrowser hover <sel>             # Hover element
acbrowser focus <sel>             # Focus element
acbrowser select <sel> <val>      # Select dropdown option
acbrowser check <sel>             # Check checkbox
acbrowser uncheck <sel>           # Uncheck checkbox
acbrowser scroll <dir> [px]       # Scroll (up/down/left/right, --selector <sel>)
acbrowser scrollintoview <sel>    # Scroll element into view
acbrowser drag <src> <dst>        # Drag and drop
acbrowser upload <sel> <files>    # Upload files
acbrowser screenshot [path]       # Screenshot (--full for full page)
acbrowser screenshot --annotate   # Annotated screenshot with numbered element labels
acbrowser screenshot --screenshot-dir ./shots    # Save to custom directory
acbrowser screenshot --screenshot-format jpeg --screenshot-quality 80
acbrowser pdf <path>              # Save page as PDF
acbrowser snapshot                # Accessibility tree with refs
acbrowser eval <js>               # Run JavaScript
acbrowser connect <port|url>      # Connect to browser via CDP
acbrowser stream enable [--port <port>]  # Start runtime WebSocket streaming
acbrowser stream status           # Show runtime streaming state and bound port
acbrowser stream disable          # Stop runtime WebSocket streaming
acbrowser close                   # Close browser (aliases: quit, exit)
acbrowser close --all             # Close all active sessions
```

## Find elements

Semantic locators with actions (`click`, `fill`, `type`, `hover`, `focus`, `check`, `uncheck`, `text`):

```bash
acbrowser find role <role> <action> [value]
acbrowser find text <text> <action>
acbrowser find label <label> <action> [value]
acbrowser find placeholder <ph> <action> [value]
acbrowser find alt <text> <action>
acbrowser find title <text> <action>
acbrowser find testid <id> <action> [value]
acbrowser find first <sel> <action> [value]
acbrowser find last <sel> <action> [value]
acbrowser find nth <n> <sel> <action> [value]
```

Options:

- `--name <name>` -- filter role by accessible name
- `--exact` -- require exact text match

Examples:

```bash
acbrowser find role button click --name "Submit"
acbrowser find label "Email" fill "test@test.com"
acbrowser find alt "Logo" click
acbrowser find first ".item" click
acbrowser find last ".item" text
acbrowser find nth 2 ".card" hover
```

## Downloads

```bash
acbrowser download <sel> <path>   # Click element to trigger download
acbrowser wait --download [path]  # Wait for any download to complete
```

Use `--download-path <dir>` (or `AGENT_BROWSER_DOWNLOAD_PATH` env) to set a default download directory. Without it, downloads go to a temporary directory that is deleted when the browser closes.

### Iframe support

Iframes are detected automatically during snapshots. `Iframe` nodes are resolved and their content is
inlined beneath the iframe element in the snapshot output. Refs assigned to elements inside iframes carry
frame context, so `click`, `fill`, and other interactions work without manually switching frames.

```bash
acbrowser snapshot -i
# @e3 [Iframe] "payment-frame"
#   @e4 [input] "Card number"
#   @e5 [button] "Pay"

# Interact directly using refs — no frame switch needed
acbrowser fill @e4 "4111111111111111"
acbrowser click @e5

# Or switch frame context for scoped snapshots
acbrowser frame @e3
acbrowser snapshot -i             # Only elements inside that iframe
acbrowser frame main              # Return to main frame
```

The `frame` command accepts element refs (`@e3`), CSS selectors (`"#my-iframe"`), or frame name/URL.

## Dialogs

```bash
acbrowser dialog accept [text]    # Accept dialog (with optional prompt text)
acbrowser dialog dismiss          # Dismiss dialog
acbrowser dialog status           # Check if a dialog is currently open
```

By default, `alert` and `beforeunload` dialogs are automatically accepted so they never block the agent. `confirm` and `prompt` dialogs still require explicit handling. Use `--no-auto-dialog` (or `AGENT_BROWSER_NO_AUTO_DIALOG=1`) to disable automatic handling.

When a JavaScript dialog (`alert`, `confirm`, `prompt`) is pending, all command responses include a `warning` field with the dialog type and message.

## Streaming

## Auth vault

```bash
acbrowser auth save <name> [opts]    # Save auth profile
acbrowser auth login <name>          # Login using saved credentials
acbrowser auth list                  # List saved profiles (names and URLs only)
acbrowser auth show <name>           # Show profile metadata (no passwords)
acbrowser auth delete <name>         # Delete a saved profile
```

Save options:

- `--url <url>` -- login page URL (required)
- `--username <user>` -- username (required)
- `--password <pass>` -- password (required unless `--password-stdin`)
- `--password-stdin` -- read password from stdin (recommended to avoid shell history exposure)
- `--username-selector <sel>` -- custom CSS selector for username field
- `--password-selector <sel>` -- custom CSS selector for password field
- `--submit-selector <sel>` -- custom CSS selector for submit button

`auth login` navigates with `load` and then waits for the username/password/submit selectors to appear before interacting. This improves reliability on SPA login pages where fields render after initial page load.

```bash
echo "pass" | acbrowser auth save github --url https://github.com/login --username user --password-stdin
acbrowser auth login github
acbrowser auth list
```

## Confirmation

When `--confirm-actions` is set, certain action categories return a `confirmation_required` response instead of executing immediately. Use `confirm` or `deny` to approve or reject the action.

```bash
acbrowser confirm <confirmation-id>  # Approve a pending action
acbrowser deny <confirmation-id>     # Deny a pending action
```

Pending confirmations auto-deny after 60 seconds.

```bash
acbrowser --confirm-actions eval,download eval "document.title"
# Returns confirmation_required with ID
acbrowser confirm c_8f3a1234
```

## Chat

Use natural language to control the browser via AI. The `chat` command translates instructions into acbrowser commands, executes them, and streams the AI response. Requires `AI_GATEWAY_API_KEY` to be set.

```bash
acbrowser chat "open google.com and search for cats"     # Single-shot instruction
acbrowser chat                                           # Interactive REPL (type quit to exit)
echo "summarize this page" | acbrowser chat              # Piped input
acbrowser -q chat "summarize this page"                  # Quiet: text only, no tool calls shown
acbrowser -v chat "fill in the login form"               # Verbose: show commands and their output
acbrowser --model openai/gpt-4o chat "take a screenshot" # Override the default AI model
acbrowser --json chat "open example.com"                 # Structured JSON output
```

Chat-specific options:

```bash
--model <name>           # AI model (or AI_GATEWAY_MODEL env, default: anthropic/claude-sonnet-4.6)
-v, --verbose            # Show tool commands and their raw output
-q, --quiet              # Show only the AI text response (hide tool calls)
```

## Batch execution

Execute multiple commands in a single invocation. Commands can be passed as quoted arguments or piped as JSON via stdin.

```bash
# Argument mode: each quoted argument is a full command
acbrowser batch "open https://example.com" "snapshot -i" "screenshot"

# With --bail to stop on first error
acbrowser batch --bail "open https://example.com" "click @e1" "screenshot"

# Stdin mode: pipe commands as JSON
echo '[
  ["open", "https://example.com"],
  ["snapshot", "-i"],
  ["click", "@e1"],
  ["screenshot", "result.png"]
]' | acbrowser batch --json
```

<table>
  <thead>
    <tr><th>Option</th><th>Description</th></tr>
  </thead>
  <tbody>
    <tr><td><code>--bail</code></td><td>Stop on first error (default: continue all commands)</td></tr>
    <tr><td><code>--json</code></td><td>Output results as a JSON array</td></tr>
  </tbody>
</table>

## Command chaining

Chain commands with `&&` in a single shell invocation. The browser persists via a background daemon, so chaining works naturally and is more efficient than separate calls:

```bash
acbrowser open example.com && acbrowser wait --load networkidle && acbrowser snapshot -i
acbrowser fill @e1 "user@example.com" && acbrowser fill @e2 "pass" && acbrowser click @e3
acbrowser open example.com && acbrowser wait --load networkidle && acbrowser screenshot page.png
```

Use `&&` when you don't need to read intermediate output. Run commands separately when you need to parse output first (e.g., snapshot to discover refs, then interact with those refs).

## Local files

Open local files (PDFs, HTML) using `file://` URLs:

```bash
acbrowser --allow-file-access open file:///path/to/document.pdf
acbrowser --allow-file-access open file:///path/to/page.html
acbrowser screenshot output.png
```

The `--allow-file-access` flag enables JavaScript to access other local files. Chromium only.
